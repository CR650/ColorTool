🧪 AllObstacle.xls 功能测试页面

📋 测试目标： 验证AllObstacle.xls文件处理功能在全新系列主题创建时的正确性
⚠️ 重要提醒：
  • 测试前请确保Unity项目文件夹中包含AllObstacle.xls文件
  • AllObstacle.xls文件必须包含Info工作表，且具有id、notes、nameID、Sort、isFilter列
  • 建议在测试前备份AllObstacle.xls文件
  • 只有创建全新系列主题时才会触发AllObstacle处理
  • 格式兼容性：系统会优先保存为XLS格式以确保与Java工具兼容
  • 重要：如果保存为XLSX格式，Java工具可能报错，需要手动转换

🔧 测试环境准备

环境检查清单

  1. 确保浏览器支持File System Access API（推荐Chrome/Edge）
  2. 准备包含以下文件的Unity项目文件夹：
    • RSC_Theme.xls
    • UGCTheme.xls
    • AllObstacle.xls（必须包含Info工作表）
  3. 确保AllObstacle.xls的Info工作表包含必要的列：id、notes、nameID、Sort、isFilter
  4. 注意：Info工作表的表头在第1行，数据从第6行开始
  5. 记录AllObstacle.xls文件中现有的最大ID和Sort值（从第6行开始的数据）

🧪 测试用例

测试用例1：全新系列主题创建（应触发AllObstacle处理）

  1. 打开ColorToolConnectRC工具主页面
  2. 选择Unity项目文件夹（包含AllObstacle.xls）
  3. 输入一个全新系列的主题名称（如"测试主题1"，确保现有主题中没有"测试主题"系列）
  4. 配置多语言ID（输入一个未使用的数字，如9999）
  5. 设置颜色配置并点击"处理主题数据"
  6. 观察控制台日志和状态提示
  7. 检查AllObstacle.xls文件的Info工作表是否新增了一行数据
预期结果：
  • 控制台显示"检测到全新系列主题，开始处理AllObstacle文件..."
  • 控制台显示"✅ AllObstacle文件处理成功"
  • 状态栏显示"AllObstacle文件处理成功"
  • AllObstacle.xls的Info工作表新增一行（在现有数据末尾），包含：
    • id: 现有数据（第6行开始）中的最大值+1
    • notes: "测试主题"（去除数字后的基础名称）
    • nameID: 9999（用户输入的多语言ID）
    • Sort: 现有数据（第6行开始）中的最大值+1
    • isFilter: 1

测试用例2：同系列主题创建（不应触发AllObstacle处理）

  1. 在测试用例1的基础上，创建同系列的新主题
  2. 输入主题名称"测试主题2"（与测试用例1同系列）
  3. 观察是否显示多语言配置面板（应该隐藏）
  4. 点击"处理主题数据"
  5. 观察控制台日志和状态提示
预期结果：
  • 多语言配置面板不显示
  • 控制台显示"非全新系列主题或多语言配置无效，跳过AllObstacle处理"
  • AllObstacle.xls文件不会被修改
  • 主题处理正常完成，但不包含AllObstacle处理

测试用例3：AllObstacle文件不存在的情况

  1. 选择一个不包含AllObstacle.xls文件的Unity项目文件夹
  2. 创建全新系列主题
  3. 观察系统行为
预期结果：
  • 控制台显示"AllObstacle.xls文件未找到，跳过处理"
  • 状态栏显示"AllObstacle.xls文件未找到，跳过AllObstacle处理"
  • 主题处理正常完成，不受AllObstacle缺失影响

测试用例4：XLS格式兼容性验证

  1. 创建全新系列主题，触发AllObstacle处理
  2. 观察保存过程中的格式选择
  3. 检查保存后的文件格式
  4. 使用Java工具验证文件兼容性：
    java -Dfile.encoding=UTF-8 -jar HerosClientBuildTools_fat.jar .\xlsx true
预期结果：
  • 控制台显示"尝试使用XLS格式（无日期处理）保存..."
  • 优先保存为XLS格式（OLE2格式）
  • Java工具能正常处理文件，不报格式错误
  • 如果保存为XLSX格式，会显示兼容性警告

测试用例5：重复多语言ID检测

  1. 查看AllObstacle.xls的Info工作表，记录一个已存在的nameID值（从第6行开始的数据中）
  2. 创建全新系列主题，使用已存在的多语言ID
  3. 观察系统行为
预期结果：
  • 控制台显示"多语言ID XXX 已存在于第 X 行，跳过处理"
  • AllObstacle.xls文件不会被修改
  • 主题处理正常完成
